AUDIT FINDINGS BY SECTION

SECTION 1 — Authentication & JWT

1.1 JWT_SECRET minimum length — FIXED (HIGH)
  File: packages/backend/src/config/env.ts line 9
  Was: Only checked for empty string. A short secret (e.g. "secret") was accepted.
  Fix: Added `if (jwtSecret.length < 32) throw new Error(...)` after existence check.

1.2 Token expiry — PASS
  Access token 15m (JWT_ACCESS_EXPIRES_IN default), refresh 30 days (REFRESH_TOKEN_TTL_DAYS).

1.3 Refresh token plaintext storage — NOTE (LOW)
  File: packages/backend/src/services/auth.service.ts line 51
  Refresh tokens stored as plaintext in DB. Risk is low (crypto.randomBytes(40) = 80-char hex, very high entropy). Hashing before storage would be ideal defence-in-depth. Added to backlog.

1.4 Logout invalidation — PASS
  revokeRefreshToken calls deleteMany; client-side clearAuth removes from localStorage.

1.5 JWT algorithm confusion — FIXED (HIGH)
  Files: auth.middleware.ts line 23, admin.middleware.ts line 30
  Was: jwt.verify(token, JWT_SECRET) — no algorithm constraint.
  Fix: jwt.verify(token, JWT_SECRET, { algorithms: ['HS256'] }) in both files.
  Risk: Without this, an attacker could forge tokens using the 'none' algorithm.

1.6 Admin DB check — PASS
  admin.middleware.ts re-queries DB for isAdmin and isActive. Does not trust JWT claim.

1.7 bcrypt rounds — PASS
  BCRYPT_ROUNDS = 12 in auth.service.ts line 11.

1.8 Timing-safe dummy compare — FIXED (MEDIUM)
  File: packages/backend/src/services/auth.service.ts line 91-95
  Was: Called bcrypt.hash(input.password, BCRYPT_ROUNDS) — hashing is slower than compare; timing slightly different from the real path.
  Fix: Changed to bcrypt.compare(input.password, DUMMY_HASH) — matches real code path timing.

NOTES FOR CLAUDE.AI:

1. Next.js CVEs (HIGH) — 4 CVEs in Next.js 14 remain unfixed because the fix requires upgrading to Next.js 15/16 which is a breaking change. CLAUDE.md currently pins Next.js 14. Migration to httpOnly cookies requires backend cookie middleware + frontend removal of localStorage reads/writes + CORS adjustments. Schedule this as a standalone sprint.

3. Account deletion missing (MEDIUM) — GDPR right-to-erasure gap. Need DELETE /api/v1/customers/me that anonymises or hard-deletes customer data and cascades to orders.

4. All CRITICAL/HIGH code-level fixes have been applied. TypeScript is clean across all 3 packages. 17 backend tests pass.

5. Bootstrap endpoint gets only the globalRateLimit (200/min). If left enabled in staging environments, someone could brute-force the secret. Consider adding authRateLimit or a specific bootstrapRateLimit, and disabling the endpoint entirely in production (leave ADMIN_BOOTSTRAP_SECRET unset).

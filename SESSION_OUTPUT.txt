NOTES FOR CLAUDE.AI:
- createRefund() lives in payments.service.ts (lines ~204-219). Signature:
    createRefund(molliePaymentId: string, amountEuroCents: number): Promise<void>
  Uses mollie.paymentRefunds.create({ paymentId, amount: { value, currency } }).
  Throws (not degrades) so that the caller in admin.service.ts can catch and decide.

- updateOrderStatus() flow for CANCELLED:
  1. Load order with lines + payment
  2. Determine if refund+restore needed (PAID/PROCESSING/SHIPPED only)
  3. If yes: call createRefund() outside the transaction; catch errors, set mollieRefundSucceeded flag
  4. Single Prisma transaction:
     - Update order status to CANCELLED
     - Write STATUS_UPDATED OrderEvent
     - Restore stock (increment by line.quantity for each OrderLine)
     - Update Payment.status → REFUNDED (success) or FAILED (mollie error)
     - Write REFUND_INITIATED or REFUND_FAILED OrderEvent
  5. Stock restore and DB writes always succeed even if Mollie fails
  6. If Mollie not configured (local dev, no API key), createRefund throws, catch logs the error, DB writes still run

- PENDING orders cancelled by admin: no refund, no stock restore (stock was never decremented for PENDING orders — the expiry job handles those).
- DELIVERED orders cancelled by admin: no refund, no stock restore (treated same as PENDING — already delivered, refund would need to be manual).
